Архитектурный справочник NuBody: Низкоуровневый рефакторинг систем внешности, разметки слоев и внутренних органов персонажей

1. Глобальный архитектурный концепт

В рамках пул-реквестов #42476 и #42715 была полностью ликвидирована старая система анатомических слотов. До рефакторинга структура тел гуманоидов описывалась иерархическим графом соединений слотов внутри легаси-прототипа type: body. Вся визуальная составляющая жестко зашивалась в расы (type: species), дублируя ресурсы в type: speciesBaseSprites, а за наложение отвечала монолитная система внешности. Это исключало возможность физической изоляции органов или конечностей при хирургическом вмешательстве и усложняло поддержку форков.

Спецификация NuBody переводит физическое строение существа на рельсы полностью децентрализованного ECS-подхода. Тело теперь представляет собой плоский рантайм-набор самостоятельных ECS-сущностей органов и конечностей, находящихся внутри контейнеров. Каждый орган содержит собственный визуальный компонент разметки слоев, а кастомизируемые маркировки (волосы, узоры, татуировки) теперь привязаны непосредственно к конкретной категории анатомического органа (Organ Category Prototype ID).

2. Критические изменения для разработчиков (Breaking Changes)

Данный раздел содержит полный перечень инфраструктурных изменений, которые кардинально влияют на совместимость кодовых баз локальных форков и сторонних репозиториев с новым апстрим-ядром.

⚠️ ПОЛНОЕ УДАЛЕНИЕ ЛЕГАСИ-КЛАССОВ И КОМПОНЕНТОВ СЕРВЕРА Класс HumanoidAppearanceSystem и связанный с ним компонент HumanoidAppearanceComponent полностью удалены из исходного кода. Любые вызовы в сторонних системах (медицинские сканеры, импланты, аномалии, системы маскировки) упадут при компиляции. Все ссылки на данные профиля должны быть переведены на HumanoidProfileComponent, а визуальное управление — на SharedVisualBodySystem.
⚠️ ИЗМЕНЕНИЕ СХЕМЫ БАЗЫ ДАННЫХ И СЕРИАЛИЗАЦИИ ПРОФИЛЕЙ Миграция базы данных 20260118084629_OrganMarkings.cs (для Postgres) добавляет в таблицу profile новую колонку organ_markings с типом данных jsonb. Из структуры десериализации профилей полностью удалены выделенные примитивные строковые поля цвета и стиля волос/бороды. Теперь волосы — это стандартные маркировки, накладываемые на зону головы (Head). Старые учетные записи игроков автоматически конвертируются при первом входе.
⚠️ КРАШ СЕРВЕРА НА ЭТАПЕ ИНИЦИАЛИЗАЦИИ РЕСУРСОВ YAML Следующие типы прототипов были полностью вырезаны из парсера движка: type: speciesBaseSprites, type: humanoidBaseSprite, type: markingPoints. Если в ваших кастомных файлах остался хоть один из этих типов, сервер упадет с ошибкой.
⚠️ УДАЛЕНИЕ СИНТАКСИСА MAGIC MIRROR И ПИKЕРОВ Старый UI-виджет SingleMarkingPicker и логика обработки одиночных сетевых сообщений вроде MagicMirrorAddSlotMessage полностью ликвидированы. Настройка внешности зеркал теперь использует унифицированную поорганную модель представления MarkingsViewModel. При обновлении форков легаси-окна кастомизации полностью перестанут функционировать.

4. Реализация клиентских систем рендеринга слоев (Content.Client)

Класс VisualBodySystem выступает главным контроллером послойной сборки RSI-состояний на холсте родительской сущности моба. Ниже приведена точная структура регистрации событий из диффа изменений: 

namespace Content.Client.Body;

public sealed class VisualBodySystem : SharedVisualBodySystem
{
    [Dependency] private readonly IConfigurationManager _cfg = default!;
    [Dependency] private readonly IPrototypeManager _prototype = default!;
    [Dependency] private readonly MarkingManager _marking = default!;
    [Dependency] private readonly SpriteSystem _sprite = default!;

    public override void Initialize()
    {
        base.Initialize();

        SubscribeLocalEvent<VisualOrganComponent, OrganGotInsertedEvent>(OnOrganGotInserted);
        SubscribeLocalEvent<VisualOrganComponent, OrganGotRemovedEvent>(OnOrganGotRemoved);
        SubscribeLocalEvent<VisualOrganComponent, AfterAutoHandleStateEvent>(OnOrganState);

        SubscribeLocalEvent<VisualOrganMarkingsComponent, OrganGotInsertedEvent>(OnMarkingsGotInserted);
        SubscribeLocalEvent<VisualOrganMarkingsComponent, OrganGotRemovedEvent>(OnMarkingsGotRemoved);
        SubscribeLocalEvent<VisualOrganMarkingsComponent, AfterAutoHandleStateEvent>(OnMarkingsState);

        SubscribeLocalEvent<VisualOrganMarkingsComponent, BodyRelayedEvent<HumanoidLayerVisibilityChangedEvent>>(OnMarkingsChangedVisibility);

        Subs.CVar(_cfg, CCVars.AccessibilityClientCensorNudity, OnCensorshipChanged, true);
        Subs.CVar(_cfg, CCVars.AccessibilityServerCensorNudity, OnCensorshipChanged, true);
    }
}

Подсистема цензуры наготы и наложение дефолтных слоев белья

При изменении CVars цензуры наготы, метод OnCensorshipChanged полностью перестраивает слои спрайтов. Генератор динамически подставляет дефолтные элементы нижнего белья из конфигурации NudityDefault группы, если игрок не выбрал кастомный вариант кастомизации персонажа: 

private IEnumerable<Marking> AllMarkings(Entity<VisualOrganMarkingsComponent> ent)
{
    foreach (var markings in ent.Comp.Markings.Values)
    {
        foreach (var marking in markings)
        {
            yield return marking;
        }
    }

    var censorNudity = _cfg.GetCVar(CCVars.AccessibilityClientCensorNudity) || _cfg.GetCVar(CCVars.AccessibilityServerCensorNudity);
    if (!censorNudity)
        yield break;

    var group = _prototype.Index(ent.Comp.MarkingData.Group);
    foreach (var layer in ent.Comp.MarkingData.Layers)
    {
        if (!group.Limits.TryGetValue(layer, out var layerLimits) || layerLimits.NudityDefault.Count < 1)
            continue;

        var markings = ent.Comp.Markings.GetValueOrDefault(layer) ?? [];
        if (markings.Any(m => _marking.TryGetMarking(m, out var proto) && proto.BodyPart == layer))
            continue; // Слой уже занят кастомной маркировкой, дефолт цензуры пропускается

        foreach (var defaultMarkingId in layerLimits.NudityDefault)
        {
            yield return new Marking(defaultMarkingId, 1);
        }
    }
}

Управление скрытием слоев одежды (HideableHumanoidLayersSystem)

Рассчитывает разницу состояний (Diff) между наборами скрытых слоев текущего и предыдущего кадров, минимизируя нагрузку на вызовы рендерера: 

private void UpdateSprite(Entity<HideableHumanoidLayersComponent> ent)
{
    // Возвращаем видимость слоям, которые вышли из зоны скрытия одеждой
    foreach (var item in ent.Comp.LastHiddenLayers)
    {
        if (ent.Comp.HiddenLayers.ContainsKey(item))
            continue;

        var evt = new HumanoidLayerVisibilityChangedEvent(item, true);
        RaiseLocalEvent(ent, ref evt);

        if (_sprite.LayerMapTryGet(ent.Owner, item, out var index, true))
            _sprite.LayerSetVisible(ent.Owner, index, true);
    }

    // Скрываем новые изолированные слои
    foreach (var item in ent.Comp.HiddenLayers.Keys)
    {
        if (ent.Comp.LastHiddenLayers.Contains(item))
            continue;

        var evt = new HumanoidLayerVisibilityChangedEvent(item, false);
        RaiseLocalEvent(ent, ref evt);

        if (_sprite.LayerMapTryGet(ent.Owner, item, out var index, true))
            _sprite.LayerSetVisible(ent.Owner, index, false);
    }

    ent.Comp.LastHiddenLayers.Clear();
    ent.Comp.LastHiddenLayers.UnionWith(ent.Comp.HiddenLayers.Keys);
}

5. Архитектура слоя MVVM и Сетка пользовательского интерфейса

Модель представления MarkingsViewModel изолирует UI-виджеты от кодовой базы систем, проверяя лимиты, ограничения по полу расы и осуществляя интеллектуальное кэширование цветовой палитры в словарь _previousColors: 

public bool TrySelectMarking(ProtoId<OrganCategoryPrototype> organ, HumanoidVisualLayers layer, ProtoId<MarkingPrototype> markingId)
{
    if (!_prototype.TryIndex(markingId, out var markingProto) || !_organData.TryGetValue(organ, out var organData) || !_organProfileData.TryGetValue(organ, out var profileData))
        return false;

    if (EnforceGroupAndSexRestrictions && !_marking.CanBeApplied(organData.Group, profileData.Sex, markingProto))
        return false;

    _markings[organ] = _markings.GetValueOrDefault(organ) ?? [];
    var organMarkings = _markings[organ];
    organMarkings[layer] = organMarkings.GetValueOrDefault(layer) ?? [];
    var layerMarkings = organMarkings[layer];

    var colors = _previousColors.GetValueOrDefault(markingId) ??
                 MarkingColoring.GetMarkingLayerColors(markingProto, profileData.SkinColor, profileData.EyeColor, layerMarkings);
    var newMarking = new Marking(markingId, colors);

    if (!_prototype.TryIndex(organData.Group, out var groupPrototype) || !groupPrototype.Limits.TryGetValue(layer, out var limits) || !EnforceLimits)
    {
        layerMarkings.Add(newMarking);
        MarkingsChanged?.Invoke(organ, layer);
        return true;
    }

    if (limits.Limit == 1 && layerMarkings.Count == 1)
    {
        layerMarkings.Clear(); // Автозамена прически при лимите слоя = 1
        layerMarkings.Add(newMarking);
        MarkingsChanged?.Invoke(organ, layer);
        return true;
    }

    if (layerMarkings.Count < limits.Limit)
    {
        layerMarkings.Add(newMarking);
        MarkingsChanged?.Invoke(organ, layer);
        return true;
    }
    return false;
}

Математика Drag & Drop позиционирования слоев

При перетаскивании слоев маркировок внутри интерфейса, маяки drop-зоны LayerDragDropBeacon используют расчёт квадрата расстояния для минимизации нагрузки на CPU за счет исключения избыточного извлечения квадратного корня: 

private bool OnContinueDrag(float frameTime)
{
    var (item, container) = _dragDropHelper.Dragged;
    LayoutContainer.SetPosition(item, UserInterfaceManager.MousePositionScaled.Position - new Vector2(32, 32));

    // Поиск ближайшего маяка drop-зоны через квадрат длины вектора
    var closestBeacon = _beacons.MinBy(beacon => 
        (UserInterfaceManager.MousePositionScaled.Position - beacon.GlobalPosition).LengthSquared());

    if (closestBeacon != _dragTarget)
    {
        _dragTarget?.UnbecomeTarget(); // Сброс старой высоты drop-зоны в float.NaN
        _dragTarget = closestBeacon;
        _dragTarget?.BecomeTarget();   // Раскрытие новой зоны: высота 64px и класс .PanelDropTarget
    }
    return true;
}

6. Справочник Интеграционных Тестов (MarkingManagerTests.cs)

Для верификации корректности миграций был разработан комплекс автоматических тестов, проверяющих усечение лимитов и конвертацию легаси-причесок гуманоидов в маркировки органа головы: 

[Test]
public async Task HairConversion()
{
    await using var pair = await PoolManager.GetServerClient();
    var server = pair.Server;
    await server.WaitIdleAsync();

    await server.WaitAssertion(() =>
    {
        var markingManager = server.ResolveDependency<MarkingManager>();
        var markings = new List<Marking>() { new("HumanHairLongBedhead2", new List<Color>() { Color.Red }) };

        // Проверяем, что легаси-волосы корректно конвертируются в орган Head архитектуры NuBody
        var converted = markingManager.ConvertMarkings(markings, "Human");

        Assert.That(converted, Does.ContainKey(new ProtoId<OrganCategoryPrototype>("Head")));
        Assert.That(converted["Head"], Does.ContainKey(HumanoidVisualLayers.Hair));
        var hairMarkings = converted["Head"][HumanoidVisualLayers.Hair];
        Assert.That(hairMarkings, Has.Count.EqualTo(1));
        Assert.That(hairMarkings[0].MarkingId, Is.EqualTo("HumanHairLongBedhead2"));
    });
    await pair.CleanReturnAsync();
}
